VRML Illustration: Creating a Panorama via the VRML Background Node


Credits: The following VRML models were developed by Helmut Dersch from the Technical University Furtwangen in Germany. The granted permission to use the models in these notes is highly appreciated.

The VRML Background node can be used to define an all-surrounding background panorama. Usually, such panoramas are specified on a sphere with infinite radius. In VRML, an infinitely large cube contained within the sphere is used. Six images are mapped onto the six planes of the cube (the images are assumed to be projected from the sphere onto the cube). To create a seamless panorama, the images (e.g., photographs) need to be "stitched" properly at the edges of the cube. In most cases, this requires special image-processing tools (see bottom of page).

The example below shows six properly stitched photographs and their relative orientation on the outspread planes of the cube.

Click on any image to view original

The images are mapped onto the six planes of the cube by using the fields frontUrl, backUrl, leftUrl, rightUrl, topUrl, and bottomUrl in the Background node: 

#VRML V2.0 utf8

Background { 
    frontUrl  "forest_f.jpg" 
    backUrl   "forest_b.jpg" 
    leftUrl   "forest_l.jpg" 
    rightUrl  "forest_r.jpg" 
    topUrl    "forest_u.jpg" 
    bottomUrl "forest_d.jpg" 
}

Advanced VRML Panorama:

The following example includes VRML geometry (a simplified helicopter) within the panorama background of the Grand Canyon and allows to navigate around the helicopter. In addition, a zoom function (plus and minus control buttons) modifies the field fieldOfView in the current Viewpoint node, thereby, creating the (false) impression of flying through the canyon. Note that changing the FOV will widen or narrow the viewing frustum, but will not change the viewer's position.
