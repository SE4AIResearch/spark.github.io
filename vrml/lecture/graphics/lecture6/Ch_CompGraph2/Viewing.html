3D VIEWING SYSTEM

In immersive VR, the center of projection (view point) corresponds to the position of the user's head. The screens inside an HMD or BOOM represent the view planes and are moving with the user's head. This entire system is being moved and oriented in space like a synthetic camera. Position and orientation of the camera can be uniquely described with respect to a fixed coordinate system by six independent parameters, the so called Six Degrees of Freedom.





  • The Six Degrees of Freedom (DOF)


    • Position (x, y, z):

      Vector from origin of fixed coordinate system to center of projection
      (also called Projection Reference Point PRP, camera point, position of viewer, view point)

    Orientation (viewing direction, view up vector):

      Vector from PRP to VRP (View Reference Point, usually center of View Plane VP) defines viewing direction

      View Up Vector (VUP) allows to rotate view plane


  • Other Parameters of 3D Viewing System


    • Size of view plane defines FOV

    View Plane Normal (VPN) allows to tilt view plane

    Front and Back Clipping Planes (FCP, BCP) define the Viewing Frustum (part of the view volume). Ultimately, only objects inside viewing frustum are displayed. 3D clipping occurs at the boundaries of the frustum.

    View plane can lie inside or outside the viewing frustum.




The semi-infinite Viewing Volume reduced to a finite Viewing Frustum
(through the introduction of a front and back clipping plane)





  • Viewing Parameters in VRML


    • The six Degrees of Freedom (DOF):
      Viewpoint node: field position and field orientation

    Field of View (FOV), size of view plane:

      Viewpoint node: field fieldOfView, default 0.785398 (45 degrees)
      view plane is always perpendicular to viewing direction (no View Plane Normal)

    Back Clipping Plane (BCP) or far clipping plane:

      NavigationInfo node: field visibilityLimit, default (0.0) = infinite visibility

    Front Clipping Plane (FCP) or near clipping plane:

      NavigationInfo node: indirectly given in first value of field avatarSize (default 0.25)
      The VRML specification recommends that the browser sets the near clipping plane to one-half of avatarSize[0].
      Note: The first value in avatarSize is the collision radius and can be thought of as the distance between the center of the avatar's body and the shoulders. The radius defines a cylinder surrounding the avatar's body. The cylinder defines the minimum width of an opening through which an avatar can squeeze.
