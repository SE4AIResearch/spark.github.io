VRML Illustration: Shadow Simulation in VRML


Credits: This VRML animation was developed by Bert Schoenwaelder and Lars Schumann at the University of Michigan Virtual Reality Laboratory.


The animation uses a regular polyhedron known as the 'Great Icosahedron' which has been modeled using an IndexedFaceSet node. The object spins, but can also be rotated manually using the mouse. Double clicking the object toggles the spin animation.

An invisible light source seems to cast a shadow of the object on the groundplate. This simulated shadow is being created as follows:


  • While the object spins, a copy of the object is being scaled continuously using
      scale   1.5   0.000001   1.5
    in a corresponding Transform node. This scaling operation reduces the y-extension of the object to almost zero (a scale factor of zero is illegal in VRML) and enlarges the x- and z- extension by a factor of 1.5.


  • The resulting 'flat' object is placed on the groundplate (separated by a small amount to avoid flickering caused by z-buffer tearing) and colored black.


  • Before flattening the shadow object, the current rotation of the original object is being applied to the shadow object, thereby, modifying the shadow outline continuously.

The enlargement of the shadow object in x- and z-direction makes it appear as if it is being cast by a point light located above the object. However, this shadow is physically not correct. A point light would require a different scale factor for each vertex depending on the original height of the vertex above the groundplate.

Setting the scale factors for the x- and z-direction to 1.0 would create a smaller, physically correct shadow cast by a directional (parallel) light shining straight down on the groundplate.

Correct shadow calculations for a general scenario (especially casting upon an arbitrary geometry and not a simple, flat groundplate) is very difficult and usually accomplished using ray tracing techniques.
